# how-to-use.md
## Yokogushi ナレッジスペースの使い方（学習・授業向け）

本ドキュメントは、Yokogushi プロジェクトを **教材としてどのように使うか** を整理したガイドである。  
「操作手順」よりも **観察の仕方・考え方** に重点を置く。

---

## 1. 基本スタンス

このプロジェクトの使い方は、次の前提に立つ。

- 正解を探す場ではない  
- まず **動いているものを観察**する  
- 分からない点は「どこで起きているか」を特定する  
- 改善や修正は Fork / Pull Request で表現する  

「できた／できない」ではなく、  
**説明できるかどうか** を重視する。

---

## 2. 授業での基本的な使い方（active）

授業で扱うのは、原則として `variants/active` 配下の実装のみとする。

### 2.1 起動する
- README に従ってローカルで起動する
- ブラウザからアクセスし、最低限以下を確認する
  - 一覧が表示される
  - 新規作成ができる
  - 詳細が見られる
  - status が切り替わる

ここでは **速く起動できること** が重要。

---

### 2.2 観察する（最重要）

起動後、次の順番で観察する。

1. **画面**
   - どの画面があるか
   - どこからどこへ遷移しているか

2. **URL**
   - 一覧・詳細・作成で URL はどう違うか
   - GET / POST の違いはどこで使われているか

3. **コード**
   - URL と対応する処理はどこに書かれているか
   - DB に触っている箇所はどこか
   - HTML を返しているのはどこか

4. **DB**
   - どのタイミングで INSERT / SELECT / UPDATE が起きるか
   - status はどこで変更されているか

---

## 3. 科目別の観察ポイント（横串）

### 3.1 HTML / CSS
- HTML は「何を表示するための構造」になっているか
- クラス名やタグ構造に意味があるか
- CSS はどこで読み込まれているか

### 3.2 JavaScript
- JavaScriptは必須か、補助か
- どのイベントで動いているか
- 無くしたら何が困るか

### 3.3 Python / 各言語
- 1リクエストはどの関数（処理）から始まるか
- 分岐（if）は何を判断しているか
- status という状態は、どこで管理されているか

### 3.4 データベース
- DB は「何を覚えているか」
- 画面に出ている情報は、DBのどの列に対応しているか
- DBが無かったら何が起きるか

### 3.5 Linux / 実行環境
- サーバはどのプロセスとして動いているか
- どのポートを使っているか
- ログはどこに出ているか

---

## 4. 課題・演習の進め方（例）

### 4.1 読解課題
- 「一覧表示は、どのファイルのどの処理で行われているか説明せよ」
- 「status 切替の処理の流れを、HTTP → DB → 画面の順で説明せよ」

### 4.2 小さな変更
- 表示項目を1つ追加する（created_atなど）
- status 表示の文言を変える
- 一覧の並び順を変える

### 4.3 PR課題
- 変更点を README に1行で説明する
- 「なぜこの変更をしたか」を PR コメントに書く
- 他人の PR をレビューする（動作・読みやすさ）

---

## 5. extensions の使い方（発展）

`variants/extensions` は **必須ではない**。

- 興味のある学生
- 比較したい外部講師
- 言語差を見たい場合

にのみ参照する。

### 見るポイント
- 同じ仕様が、どのように違う形で書かれているか
- どこが簡単になり、どこが複雑になったか
- 自分ならどの実装を選ぶか、その理由

---

## 6. NGな使い方

- コピペして「完成させる」ことを目的にする
- フレームワークの機能暗記に寄る
- 1つの実装だけ見て終わる
- 「この言語が最強」という結論にする

---

## 7. ゴール（到達イメージ）

このプロジェクトを通して目指すのは、次の状態である。

- Webアプリの処理の流れを **言葉で説明できる**
- 技術が変わっても、何が同じで何が違うか分かる
- 小さな変更を **自分の判断で提案できる**
- 他人のコードを読んで、意見を述べられる

Yokogushi は、  
**技術を学ぶ場であると同時に、考え方を鍛える場**である。
