# overview.md  
## ナレッジスペース（Yokogushi Project）概要

このプロジェクトは、  
**Yahoo!知恵袋のような「質問と知識の集積」をモデルにしたナレッジスペース**を、  
教育目的でシンプルに実装・公開する取り組みです。

ただし、本プロジェクトが目指すのは  
「正しい答えを早く得ること」ではありません。

---

## このナレッジスペースが目指すもの

一般的な Q&A サービスでは、

- 質問する  
- 誰かが答える  
- 解決したら終わる  

という流れになりがちです。

本プロジェクトでは、そこに次の視点を加えます。

- なぜその問題が起きたのか  
- どこを観察すれば状況が把握できるのか  
- どのレイヤー（HTML / JS / Python / DB / Linux）の問題か  
- どのように整理すれば、次に再利用できる知識になるか  

つまり、  
**「答え」よりも「たどり着くまでの思考と構造」を残すこと**を重視します。

---

## チケットからナレッジへ

このナレッジスペースでは、投稿される内容を次のように捉えます。

- 最初は「困りごと」「質問」として投稿される  
- 内容が整理されると「対応中のチケット」になる  
- 解決・整理されたものは「ナレッジ」として蓄積される  

見た目は掲示板に近いですが、  
実際には **学習や作業を前に進めるための記録**です。

---

## 教材としての位置づけ

このプロジェクトは、完成品を提供する教材ではありません。

- コードは **読むためのもの**  
- 改造・改善・議論されることを前提とする  
- あえて未完成・未最適な部分を残す  
- 正解や最終形を示さない  

学生は Fork や Pull Request を通じて、  
既存のコードや構成を読み、自分の理解を変更として表現します。

---

## 技術横断（横串）の考え方

本プロジェクトは、以下の技術を  
**1つのシステムとして横断的に観察する対象**として設計されています。

- HTML / CSS：構造と表現  
- JavaScript：振る舞いと非同期処理  
- Python：業務ロジック  
- データベース：状態と履歴  
- Linux：実行環境、プロセス、ログ  

個々の技術を単独で学ぶのではなく、  
**どの技術が、どの役割で使われているのか**を理解することが狙いです。

---

## 情報技術科との関係

本プロジェクトは高度専門士課程の実践教材であると同時に、  
情報技術科の学生にとっても **観察教材**として位置づけられます。

- 成果物を閲覧できる  
- システム全体像を把握できる  
- 次の段階で何が求められるかを具体的に知る  

といった形で、学年・学科を越えた参照対象となることを想定しています。

---

## 注意事項

- 本プロジェクトは学習用です  
- 正確性や完全性を保証するものではありません  
- 本番運用や商用サービスを目的としていません  

未完成さや曖昧さも含めて、  
**観察・議論・改善の対象**としてください。

---

## まとめ

このナレッジスペースは、

- Yahoo!知恵袋のように「誰かの困りごと」から始まり  
- OSSのように「みんなで育てる知識」へと変わっていく  

**学習のための共有知の場**を目指しています。
